Instance settings

PageSpeed Tester

Times in the dashboard and project cron schedules use this timezone. Scheduled runs are checked every 15 minutes; disable them globally while keeping manual triggers available.

Off by default — smaller reports and lower R2 usage. When enabled, JSON files are much larger; R2 storage fills faster and may incur Cloudflare charges. Screenshots appear in the report detail view for runs started after you save this setting.

GitHub

Repository where Lighthouse runs via Actions. Set owner and repo to this instance’s repo — the GH_PAT secret must have access to it. Upstream fields apply when syncing template updates (see Upstream sync below).

Session & domains

Shared parent domain for login cookies across dashboard and API (e.g. .kunde.de). Leave empty for *.pages.dev previews.

Upstream sync

Merge updates from the upstream template into your GitHub repository (the owner/repo above). Works for repos created via Use this template or as a fork. Requires GH_PAT with Contents: Read and write on your repo and Pull requests: Read and write for template sync (cross-repo PR); Cloudflare redeploys Worker/Pages after a successful push to main. On merge conflicts, resolve manually on GitHub or with git.

Click Refresh status to compare your repo with upstream (GitHub API).
